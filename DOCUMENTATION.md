## Обзор проекта

**Система анализа ЭКГ с AI** — это веб-приложение для медицинской диагностики, которое использует искусственный интеллект для анализа электрокардиограмм (ЭКГ) и генерации медицинских отчетов на русском языке.

### Ключевые возможности

- 📊 **Загрузка и анализ ЭКГ**: Поддержка форматов PDF, PNG, JPG, JPEG, WEBP
- 🤖 **AI-анализ**: Использование Google Gemini 2.5 Flash 
- 📝 **Генерация отчетов**: Автоматическое создание медицинских заключений с оценкой риска
- 🎤 **Голосовой ассистент**: Консультации с AI-кардиологом через голосовой интерфейс
- 👤 **Управление пациентами**: Многошаговая форма сбора анамнеза
- 🔒 **Аутентификация**: Безопасный доступ с управлением правами пользователей

### Целевая аудитория

- Врачи-кардиологи
- Медицинские учреждения
- Пациенты для первичной консультации

## Архитектура системы

```
┌─────────────────────────────────────────────────────────────┐
│                      Frontend (React + Vite)                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │ ECG Analysis │  │ Voice        │  │ Auth Pages   │     │
│  │ Form         │  │ Assistant    │  │              │     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    Supabase Backend                         │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │ PostgreSQL   │  │ Storage      │  │ Auth         │     │
│  │ Database     │  │ (ECG Files)  │  │ System       │     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │          Edge Functions (Deno/TypeScript)            │  │
│  │  • analyze-ecg      • generate-report                │  │
│  │  • voice-assistant                                   │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  Google Gemini 2.5 Flash API                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Поток данных
1. **Загрузка ЭКГ**: Пользователь загружает файл ЭКГ → Сохраняется в Storage → Создается запись в `ecg_scans`
2. **Анализ**: Edge Function `analyze-ecg` → Gemini API → Gemini анализирует изображение → Результат сохраняется в БД
3. **Генерация отчета**: Edge Function `generate-report` → Gemini API → Gemini создает отчет → Сохраняется в `medical_reports`
4. **Голосовой ассистент**: Web Speech API → Edge Function `voice-assistant` → Gemini API → Ответ озвучивается

### Компоненты формы

#### `ECGAnalysisForm.tsx`
Многошаговая форма анализа ЭКГ.

**Шаги**:
1. `StepOne`: Загрузка файла ЭКГ
2. `StepTwo`: Демографические данные (возраст, пол, рост, вес)
3. `StepThree`: Текущие жалобы (чекбоксы + свободное поле)
4. `StepFour`: Медицинский анамнез и факторы риска

---

#### `StepOne.tsx`
Компонент загрузки файла ЭКГ.
**Функциональность**:
- Drag & drop область
- Валидация типов файлов (PDF, PNG, JPG, JPEG, WEBP)
- Превью загруженного файла
- Загрузка в Supabase Storage
- Создание записи в `ecg_scans`
- Вызов `analyze-ecg` Edge Function
---

#### `StepTwo.tsx`
Сбор демографических данных пациента.
**Поля**:
- Возраст (число)
- Пол (радиокнопки: мужской/женский)
- Рост в см (число)
- Вес в кг (число)
---

#### `StepThree.tsx`
Сбор информации о текущих жалобах.
**Чекбоксы**:
- Боль в груди
- Одышка
- Сердцебиение
- Головокружение
- Усталость
- Обморок
- Отеки
- Другое
**Дополнительно**: Текстовое поле для детального описания жалоб
---

#### `StepFour.tsx`
Медицинский анамнез и факторы риска.
**Поля**:
- История болезней (текстовое поле)
- Факторы риска (чекбоксы):
  - Курение
  - Диабет
  - Гипертония
  - Высокий холестерин
  - Семейная история сердечных заболеваний
  - Ожирение
---

## Безопасность

### Row Level Security (RLS)
**Все таблицы защищены RLS-политиками**:
### Аутентификация

- **Метод**: Email + Password
- **Auto-confirm**: Включено (для разработки)
- **Защищенные роуты**: Все страницы кроме Login/Register

### Хранение файлов

- **Bucket**: `ecg-files` (публичный)
- **Изоляция**: Файлы организованы по `user_id`
- **Очистка**: Рекомендуется настроить автоудаление старых файлов

### Best Practices

- ✅ Используйте RLS на всех таблицах
- ✅ Проверяйте `auth.uid()` в Edge Functions
- ✅ Валидируйте входные данные с Zod
- ✅ Не храните чувствительные данные на клиенте
- ✅ Регулярно обновляйте зависимости
